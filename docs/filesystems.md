Linux code philosophy is "everything in linux is a file"
You have actual files, process information, memory-backed files etc everything described as files.
These are filesystem types.

```bash
df -T
```


you'll see types like:

```text
ext4
proc
sysfs
tmpfs
overlay
xfs
```

Each type tells the kernel **how to interpret data behind that mount point**.

---

### ext4

Normal disk filesystem.

Usually your SSD/HDD.

Example:

```text
/dev/nvme0n1p3 -> ext4 -> /
```

Files are physically stored on disk.

```text
ext4
├── stores files
├── stores directories
├── stores permissions
└── survives reboot
```

Most of:

```text
/home
/etc
/usr
```

ultimately live on ext4 (or xfs, btrfs, etc.).

---

### proc

Mounted at:

```text
/proc
```

This is a virtual filesystem generated by the kernel.

There are no actual files on disk.

Example:

```bash
cat /proc/self/status
```

The kernel generates that content when you read it.

```text
Process
   |
read()
   |
Kernel creates text
   |
Returns text
```

---

### sysfs

Mounted at:

```text
/sys
```

Another virtual filesystem.

Exposes kernel and device information.

Example:

```bash
ls /sys/class/net
```

shows network interfaces.

```text
eth0
lo
```

These aren't normal files.

They are views into kernel objects.

---

### tmpfs

Mounted commonly at:

```text
/tmp
/run
/dev/shm
```

RAM-backed filesystem.

```text
tmpfs
    ↓
Memory
```

Creating a file:

```bash
echo hello > /tmp/a.txt
```

stores it in RAM, not on disk.

After reboot everything is gone  

because RAM is cleared.

---

### devtmpfs

Mounted at:

```text
/dev
```

Contains device files.

Examples:

```text
/dev/null
/dev/zero
/dev/random
/dev/sda
```

Generated by the kernel.

These represent devices, not regular files.

---



### xfs

Alternative to ext4.

Often used on servers.

```text
xfs
├── disk filesystem
├── persistent
└── high-performance
```

Same purpose as ext4.

---

## Why are they all called filesystems?

Because Linux exposes everything through the filesystem interface.

A program uses:

```c
open()
read()
write()
close()
```

regardless of whether it's reading:

```text
/home/file.txt     (ext4)
/proc/cpuinfo      (proc)
/sys/...           (sysfs)
/dev/null          (devtmpfs)
```

The kernel routes the request to the correct filesystem implementation.

---

```text
ext4      -> real files on disk
proc      -> process information
sysfs     -> kernel/device information
tmpfs     -> memory-backed files
```
